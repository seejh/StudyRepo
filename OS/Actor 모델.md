"객체 + 자체 스레드 + 메시지 큐" 형태로 구성된 동시성 프로그래밍 모델.<br/>
핵심 철학은 공유 메모리를 쓰지 않고, 모든 상호작용은 메시지로만 하는 것.<br/>

각 actor는 독립된 존재이며, 서로에게 메시지를 보내서만 소통.<br/>
actor 내부 상태는 오직 그 actor만 수정할 수 있다.<br/>
다른 스레드는 절대 해당 상태에 직접 접근하지 않는다.<br/>

actor 모델
장점
락 없음 -> 성능 증가
데드락 없음
CPU 코어 활용을 균등하게 분산 가능

단점
actor 간 메시지 왕래가 많으면 병목
actor가 너무 커지면 단일 스레드 부하 증가
