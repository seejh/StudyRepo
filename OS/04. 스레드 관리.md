# 스레드
Light Weight Process, 자원&제어 에서 제어 요소만 가지는.
프로세서(CPU)활용의 기본 단위

구성 요소
Thread ID
Register Set(PC, SP 등) - 제어하기 위한
Stack(local data) - 작업 영역

제어 요소 외 코드, 데이터 및 자원들은 프로세스 내 다른 스레드들과 공유

어떤 프로세스가 있고 그 프로세스가 할당 받은 메모리 공간을 공유해서 사용.
공용 메모리 공간에서 자원에 해당하는 공간은 공유해서 사용하고 제어 부분은 그 공간에서 구분해서 사용.


스레드의 장점

자원 공유
프로세스는 번갈아(컨텍스트 스위치 발생, 커널의 개입) 사용해야 한다.
스레드는 동시에 사용 가능하다.(컨텍스트 스위치 발생 X, 커널의 개입을 피할 수 있다.)

경제성
프로세스의 생성, 컨텍스트 스위치에 비해 효율적

멀티 프로세서 활용
스레드는 CPU 사용 단위, 여러 개를 동시에 사용할 수 있다.
-> 병렬 처리를 통해 성능 향상.

사용자 응답성
일부 스레드의 처리가 지연되어도, 다른 스레드는 작업을 계속 처리 가능.
