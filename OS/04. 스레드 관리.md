
## 스레드
프로세스가 어떠한 자원을 할당 받고 제어한다고 생각할 때 스레드는 제어에 대한 부분을 담당한다.<br/>

한 프로세스를 위한 공간이 잡혀 있고 리소스(힙, 데이터, 코드) 영역은 공유.
각 스레드 별로 본인 작업 공간(스택)이 따로 존재.
코드 영역에 소스 코드가 있고 각 스레드의 PC가 실행할 코드를 가리킴.

## 그래서 스레드란?
프로세서(CPU) 활용의 기본 단위.<br/>
제어 요소 외 코드, 데이터 자원들은 프로세스 내 다른 스레드들과 공유.<br/>
전통적 프로세스 = 단일 스레드 프로세스.<br/>

구성요소
* 스레드ID
* Register Set(PC, SP 등)
* Stack

## 스레드 장점(vs 프로세스)
### 1. 가벼운 생성과 전환
스레드는 프로세스보다 생성 비용이 훨씬 적다. 새 프로세스를 만들 때는 메모리 공간 전체를 복사해야 하지만,
스레드는 같은 프로세스 내에서 실행 흐름만 추가하면 되기 때문이다. <br/>

메모리 주소 공간을 전환할 필요가 없기 때문에 컨텍스트 스위칭도 더 빠르다.
### 2. 효율적인 자원 공유
같은 프로세스 내 스레드들은 메모리 공간(코드, 데이터, 힙)을 공유한다.
프로세스 간 통신(IPC)처럼 복잡한 메커니즘 없이도 데이터를 쉽게 주고 받을 수 있어서 협업이 필요한 작업에
유리하다.

### 3. 메모리 효율성
여러 스레드가 같은 메모리 공간을 사용하므로, 동일한 작업을 여러 프로세스로 처리하는 것보다0
