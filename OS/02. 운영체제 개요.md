# OS?
컴퓨터 하드웨어를 효율적으로 관리해서 사용자 또는 응용 프로그램에 서비스를 제공하는 녀석.

## 운영체제의 역할
### 1. User Interface(편리성)
사용자가 시스템을 편하게 사용할 수 있도록<br/>
CUI, GUI 등 제공<br/>

### 2. Resource Management(효율성)
HW Resource: processor, memory, I/O devices, ...<br/>
SW Resource: file, application, message, signal, ...<br/>

### 3. Process and Thread Management
### 4. System Management(시스템 보호)

컴퓨터 하드웨어가 있고 그 위에 OS가 존재하여 OS가 응용 프로그램과 사용자들에게 서비스를 제공한다.<br/>

사용자가 하드웨어에 직접 접근하면 리스크가 크므로 OS에 요청을 하여 처리하는 방식을 사용하고
사용자가 OS에 요청을 하는 통로(함수)를 시스템 콜 인터페이스라고 한다.<br/>

## 운영체제의 구분
* 동시 사용자 수
* 동시 실행 프로세스 수
* 작업 수행 방식

## 운영체제의 구조
* 커널<br/>
  OS의 핵심 부분. (계속 사용될 수 있게 메모리에 상주.)<br/>
  
* 유틸리티(커널을 제외한 나머지)<br/>
  UI 등 서비스 프로그램.<br/>
  비상주 프로그램.<br/>

## 운영체제의 기능
* 프로세스(Process) 관리
* 프로세서(Processor) 관리
* 메모리(memory) 관리
* 파일(File) 관리
* 입출력(I/O) 관리
* 보조 기억 장치 및 기타 주변장치 관리 등
