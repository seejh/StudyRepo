# 정적 라이브러리
### 정적 라이브러리 생성
```
1. 프로젝트 생성
비주얼 스튜디오 -> 새 프로젝트 생성 -> 정적 라이브러리(MyMathLib)

// MyCalculator.h
class MyCalculator {
public:
  void Add(int value);
  void Sub(int value);
  void GetTotal() const;
private:
  int m_total;
};

// MyCalculator.cpp
void MyCalculator::Add(int value) { m_total += value; }
void MyCalculator::Sub(int value) { m_total -= value; }
void MyCalculator::GetTotal() const { return m_total; }

// pch.h
#ifndef PCH_H
#define PCH_H
#include<iostream>
#endif
```

### 정적 라이브러리 링크
본 프로젝트에서 정적 라이브러리를 링크하려면 아래 3단계를 거쳐야 한다.
```
1. 헤더 파일 경로
프로젝트 속성 -> c/c++ -> 일반 -> 추가 포함 디렉토리 -> 헤더 파일 경로
2. 라이브러리 경로
프로젝트 속성 -> 링커 -> 일반 -> 추가 라이브러리 디렉토리 -> .lib 파일이 있는 폴더
3. 사용할 라이브러리 명시
프로젝트 속성 -> 링커 -> 입력 -> 추가 종속성 -> .lib 파일
또는 #pragma comment(lib, ".lib 파일") 으로도 가능하다.
```



#### 정적 라이브러리 PCH
라이브러리 내부 소스들만 PCH의 이점을 누리고, 외부 프로젝트에는 아무런 영향을 주지 않는 것이 정상

#### PCH
빌드 시간 단축, 중복 컴파일 방지


iostream, vector, windows.h 같은 무겁고 자주 안 바뀌는 헤더를 pch.h에 몰아 넣고 빌드하면
컴파일러가 이 헤더들을 딱 한 번만 미리 빌드해둔다.
라이브러리를 컴파일할 때 매번 수만 줄의 헤더를 읽지 않고 미리 빌드된 파일을 읽을 수 있기 때문에
라이브러리 빌드 속도가 빨라진다.

메인 프로젝트에 pch.h를 강제하면 안된다.

배포용 헤더에는 pch 코드를 넣으면 안 된다.

### 연결
```
명시적 연결 (Explicit Linking), 암시적 연결 (Implicit Linking)

명시적 연결
런타임에 필요한 순간에 직접 라이브러리를 로드하고 해제하는 방식

라이브러리 로드, LoadLibrary()
함수 주소 가져오기, GetProcAddress()
라이브러리 해제, FreeLibrary()

-----------------------------------

암시적 연결
.h, .lib, .dll
프로그램이 시작될 때 운영체제가 라이브러리를 자동으로 메모리에 올리는 방식

소스 코드 상단에 #include로 헤더 파일 추가, 프로젝트 설정에서 .lib 파일을 종속성으로 연결

프로그램 실행 파일과 같은 폴더나 시스템 경로에 .dll 파일이 반드시 존재해야 한다.
```
