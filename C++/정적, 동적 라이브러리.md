### 연결
```
헤더 파일, 라이브러리 파일을 제공해야 하는데 이 2개를 제공하기 위해 3가지 단계를 거쳐야 한다.
헤더파일 경로, 라이브러리 파일 경로, 라이브러리 명시

1. 헤더 파일 경로
프로젝트 속성 -> c/c++ -> 일반 -> 추가 포함 디렉토리 -> 헤더 파일 경로 지정

2. 라이브러리 경로
프로젝트 속성 -> 링커 -> 일반 -> 추가 라이브러리 디렉토리 -> .lib 파일이 있는 폴더 지정

3. 라이브러리 명시
프로젝트 속성 -> 링커 -> 입력 -> 추가 종속성 -> .lib 파일 이름 지정

3번의 경우 아래와 같이도 가능
#pragma comment(lib, "라이브러리명")
```

### 연결
```
명시적 연결 (Explicit Linking), 암시적 연결 (Implicit Linking)

명시적 연결
런타임에 필요한 순간에 직접 라이브러리를 로드하고 해제하는 방식

라이브러리 로드, LoadLibrary()
함수 주소 가져오기, GetProcAddress()
라이브러리 해제, FreeLibrary()

-----------------------------------

암시적 연결
.h, .lib, .dll
프로그램이 시작될 때 운영체제가 라이브러리를 자동으로 메모리에 올리는 방식

소스 코드 상단에 #include로 헤더 파일 추가, 프로젝트 설정에서 .lib 파일을 종속성으로 연결

프로그램 실행 파일과 같은 폴더나 시스템 경로에 .dll 파일이 반드시 존재해야 한다.
```
