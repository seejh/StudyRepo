# PCH (Precompiled Header, 미리 컴파일된 헤더)
### 사용 목적 및 장점
PCH를 사용하면 빌드 속도가 빨라진다.<br/>

* 빌드 시간 단축
  * 대형 헤더 파일들을 매번 컴파일하지 않고 미리 컴파일해 두어 빌드 속도를 높인다.
* 중복 컴파일 방지
  * 여러 .cpp 파일이 공통으로 포함하는 헤더(windows.h, vector)를 한 번만 컴파일한다.

### 미리 컴파일된 헤더에 주로 포함되는 헤더 파일 속성
* 자주 변경 되지 않음
* 여러 곳에서 참조됨

### 단점
메모리, 디스크 사용량 증가<br/>

일반적으로 컴파일러는 .cpp 파일을 하나씩 처리할 때마다 메모리를 비우고 새로 시작한다.
하지만 pch를 쓰면 컴파일러가 해당 내용을 컴파일하는 동안 메모리에 상주시킨다.

### 결론
프로그램 실행과는 관계없는, 컴파일(빌드)하는 과정에서 메모리와 디스크 공간을 저당 잡히는 대신 컴파일
속도를 얻는 트레이드 오프.
