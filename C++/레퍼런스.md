# 1. 레퍼런스 반환
일반 변수 반환하는 것과 레퍼런스로 반환하는 것의 차이

```c++
int t1(); // 값 반환
int& t2(); // 레퍼런스 반환

// 값 반환
int x = 10;
int t1() { return x; }

t1() = 20; // 컴파일 에러

// 레퍼런스 반환
int x = 10;
int& t2() { return x; }

t2() = 20; // OK
```
* 값 반환
  * f() = 임시값(rvalue)
  * 수정할 수 있는 대상 아님
 
* 레퍼런스 반환
  * t2()는 x의 다른 이름(=별칭, alias)
  * 좌변(lvalue)로 사용 가능

반환(return)할 때 레퍼런스 타입으로 반환하면 복사 비용을 아낄 수 있다.<br/>
그래서 stl 컨테이너는<br/>
조회: const T&,<br/>
수정: T&<br/>
위와 같이 사용한다. 사용 시 주의할 점은 레퍼런스를 반환 후에 객체가 살아있음을 보장되는 환경에서 사용해야 한다.
