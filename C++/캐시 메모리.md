# 캐시
컴퓨터 분야에 사용되는 용어로 데이터를 미리 복사해 놓는 임시 장소를 가리킨다.
데이터가 본래 저장되어 있던 장소보다 접근 시간이 더 빠른 별도의 장소에 데이터를 미리 복사해 놓으면
계산이나 접근 시간 없이 더 빠른 속도로 데이터에 접근할 수 있다.

# 캐시 메모리(Cache Memory)
아래는 메모리 계층 구조에 대한 그림.<br/>
레지스터, 캐시, 메인 메모리 등이 있고 나열된 순서대로 속도가 빠르고 CPU에 가깝고 비싸고 용량이 적다.<br/>

그러니까 캐시는 필요한 데이터를 멀리가서 찾을 필요없이 가까운 장소에 미리 필요할만한 데이터를 예상해서 갖다 놓는 방식으로
이 예상이 적중할 확률을 높이기 위해 데이터 지역성이라는 것을 사용한다.<br/>

* **Cache Hit**
  * CPU가 참조하려는 데이터가 캐시 메모리에 있는 경우
* **Cache Miss**
  * CPU가 참조하려는 데이터가 캐시 메모리에 없음
  * ***Compulsory Miss***
    * 해당 데이터에 대한 요청 자체가 처음.
  * ***Capacity Miss***
    * 요청 받아서 들어왔으나 캐시 크기 한계로 인해서 빠짐
    * 그 후 다시 요청됬는데 없는 경우.
  * ***Conflict Miss***
    * 캐시 용량은 충분, 매핑 방식으로 인한 충돌
    * direct-mapped, low-associativity에서 발생.
  * ***Coherence Miss***

## 데이터 지역성(Data Locality)
### 공간 지역성
사용된 데이터의 인접한 데이터가 곧 사용될 가능성이 높다.<br/>
ex)<br/>
* 배열
* 구조체
* 순차적 코드 실행
### 시간 지역성
최근 사용된 데이터가 또 사용될 가능성이 높다.<br/>
ex)<br/>
* 반복문
* 함수 호출
