# 네트워크 부하
```
1. 전송 횟수 폭발 (N의 2제곱)
내 주변에 1000명이 있다고 1000명의 위치 패킷을 다 나에게 보내면 네트워크가 터진다.
서버가 틱마다 주변 정보를 보내줄 때 전송 횟수는 [주변 유저수 * 받는 유저 수]가 된다.
유저가 10명 : 10명 * 10명 = 100번
유저가 100명 : 10000번
유저가 1000명 : 100'0000번
겨우 100ms(1틱) 사이에 서버는 백만 번의 패킷을 만들어 쏴야 한다.(1초 10틱으로는 천만 번)
아무리 성능 좋은 서버라도 네트워크 대역폭이 순식간에 바닥난다.

2. 패킷 크기 압박
패킷 하나를 작게 잡아서 100바이트라고 할 때, 1000명이 한 자리에 모여서 서로의 정보를 다 받는다면
나 한 명이 받는 데이터: 1000명 * 100B = 100KB (매 1틱마다), 1초 10틱 동안 받는 데이터 = 1MB/s
이거를 1000명에게 보낸다고 하면 1GB/s의 업로드 속도가 필요하다. 일반적인 서버 회선으로 감당할 수 없는 수치.

그래서 mmorpg 등은...
1. 관심 영역(AOI, Area Of Interest)
내 캐릭 기준 반경 어느 정도 안에 있는 사람 정보만 보낸다.
2. 가시성 우선 순위
해당 반경 안에 500명이 있다면? 나와 파티인 사람, 나를 공격하는 사람 등의 정보를 최우선적으로 보내고
모르는 행인의 정보는 전송 주기를 늦추거나 생략.
3. 그리드 시스템
월드를 바둑판처럼 쪼개, 내가 속한 칸과 인접한 칸에 있는 유저끼리만 패킷을 주고 받게 필터링
```
