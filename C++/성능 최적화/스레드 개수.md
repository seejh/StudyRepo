void test(int threadCount, int seconds) {
  totalWork = 0;
  vector<thread> threads;

  auto start = chrono::steady_clock::now();

  for (int i = 0; i < threadCount; i++)
    threads.emplace_back(cpuWork, seconds);

  for (auto& t : threads)
    t.join();

  auto elapsed = chrono::duration_cast<chrono::milliseconds>
    (chrono::steady_clock::now() - start).count();

  cout << "스레드 수: " << threadCount << " | 총 작업량: " << totalWork << " | 시간: " << elapsed << "ms" << endl;

  int main() {
    int seconds = 5;

    for (int i = 1; i <= 20; i++)
      test(i, seconds);
  }

### 구조

### 뭐가 좋은가
CPU는 항상 100% 바쁜 것이 아니다.
```
* 일반 코어 (스레드 1개)
사이클: 1     2    3   4    5    6
스레드: 연산 대기 대기 연산 대기 연산
              캐시 미스로 메모리 기다림, 실행 유닛 놀고 있음

* 하이퍼스레딩 (스레드 2개)
사이클:  1    2   3    4    5    6
스레드1:연산 대기 대기 연산 대기 연산
스레드2:     연산 연산      연산
             1이 대기하는 동안 2가 실행 유닛 사용
```
A가 메모리 대기 등으로 쉬는 타임을 B가 채운다.

### 한계점
두 스레드가 공유하는 자원이 있기 때문이다.
```
* 공유되는 것
실행 유닛, L1/L2 캐시, 메모리 버스

* 공유되지 않는 것
레지스터, 프로그램 카운터
```
두 스레드가 동시에 같은 실행 유닛을 원하면 결국 하나가 기다려야 한다. 위에서 실행한 예제는
정말 단순한 예제로 cpu 작업만 거의 하는 작업으로, 실행 유닛을 안 쓰는 틈이 거의 없어서
