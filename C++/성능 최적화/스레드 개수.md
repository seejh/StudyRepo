## cpu 바운드 작업
```c++
std::atomic<long long> gTotalWorks = 0;

void CpuWork(int seconds) {
    // 5초 후 시간 (5초를 수행하기 위해)
    auto end = std::chrono::steady_clock::now() + std::chrono::seconds(seconds);
    long long count = 0;
    
    // 단순 정수 연산
    while (std::chrono::steady_clock::now() < end) 
        count++;  // 순수 CPU 연산

    // 
    gTotalWorks += count;
}

void Test(int threadCount, int seconds) {
    gTotalWorks = 0;
    std::vector<std::thread> threads;

    // 시작
    auto start = std::chrono::steady_clock::now();

    // 스레드 생성, 수행
    for (int i = 0; i < threadCount; i++)
        threads.emplace_back(CpuWork, seconds);
    for (auto& t : threads)
        t.join();

    // 끝
    auto elapsed = std::chrono::duration_cast<std::chrono::milliseconds>
        (std::chrono::steady_clock::now() - start).count();

    std::cout << "스레드 수: " << threadCount
        << " | 총 작업량: " << gTotalWorks
        << " | 시간: " << elapsed << "ms\n";
}

int main() {
    int seconds = 5;
    
    for (int i = 1; i <= 20; i++)
        Test(i, seconds);
}
```

## 메모리 바운드

```c++
std::atomic<long long> totalWork = 0;

// 배열을 전역으로 공유 (한 번만 할당)
const int SIZE = 1024 * 1024 * 16;  // 64MB
std::vector<int> sharedData(SIZE);

void memoryBoundWork(int seconds) {
    std::mt19937 rng(std::random_device{}());
    std::uniform_int_distribution<int> dist(0, SIZE - 1);

    auto end = std::chrono::steady_clock::now() + std::chrono::seconds(seconds);
    long long count = 0;

    while (std::chrono::steady_clock::now() < end) {
        int idx = dist(rng);
        // 읽기만 수행 (쓰기 시 false sharing 문제 발생 가능)
        volatile int val = sharedData[idx] + sharedData[(idx + 1) % SIZE];
        (void)val;
        count++;
    }

    totalWork += count;
}

int main() {
    std::iota(sharedData.begin(), sharedData.end(), 0);

    int seconds = 5;
    for (int i = 1; i <= 20; i++) {
        totalWork = 0;
        std::vector<std::thread> threads;

        auto start = std::chrono::steady_clock::now();

        for (int j = 0; j < i; j++)
            threads.emplace_back(memoryBoundWork, seconds);

        for (auto& t : threads)
            t.join();

        auto elapsed = std::chrono::duration_cast<std::chrono::milliseconds>
            (std::chrono::steady_clock::now() - start).count();

        std::cout << "스레드 수: " << i
                  << " | 총 작업량: " << totalWork
                  << " | 시간: " << elapsed << "ms\n";
    }
}
```

## i/o 바운드 작업
i/o 바운드는 대기 시간이 핵심이다.
cpu 바운드와 달리 스레드가 많아도 cpu를 놓고 싸우지 않어서 코어 수 제한을 덜 받는다.
공식처럼 쓰이는 계산식이 있다.



```
적정 스레드 수 = 코어 수 * (1 + 대기 시간 / 처리 시간)
예제:
코어 8개, I/O 대기 시간 90ms, 실제 처리 시간 10ms

std::atomic<long long> totalWork = 0;

void ioBoundWork(int seconds) {
    auto end = std::chrono::steady_clock::now() + std::chrono::seconds(seconds);
    long long count = 0;

    while (std::chrono::steady_clock::now() < end) {
        // 실제 처리 (CPU 작업)
        volatile int x = 0;
        for (int i = 0; i < 1000; i++) x += i;  // 1ms 분량 연산

        // I/O 대기 흉내 (처리 1ms : 대기 9ms = 1:9 비율)
        std::this_thread::sleep_for(std::chrono::milliseconds(9));

        count++;
    }

    totalWork += count;
}

int main() {
    int seconds = 10;
    for (int i = 1; i <= 40; i++) {
        totalWork = 0;
        std::vector<std::thread> threads;

        auto start = std::chrono::steady_clock::now();

        for (int j = 0; j < i; j++)
            threads.emplace_back(ioBoundWork, seconds);

        for (auto& t : threads)
            t.join();

        auto elapsed = std::chrono::duration_cast<std::chrono::milliseconds>
            (std::chrono::steady_clock::now() - start).count();

        std::cout << "스레드 수: " << i
                  << " | 총 작업량: " << totalWork
                  << " | 시간: " << elapsed << "ms\n";
    }
}
```
코어 수를 훨씬 넘어서도 작업량이 계속 증가.

# 하이퍼스레딩
### 구조

### 뭐가 좋은가
CPU는 항상 100% 바쁜 것이 아니다.
```
* 일반 코어 (스레드 1개)
사이클: 1     2    3   4    5    6
스레드: 연산 대기 대기 연산 대기 연산
              캐시 미스로 메모리 기다림, 실행 유닛 놀고 있음

* 하이퍼스레딩 (스레드 2개)
사이클:  1    2   3    4    5    6
스레드1:연산 대기 대기 연산 대기 연산
스레드2:     연산 연산      연산
             1이 대기하는 동안 2가 실행 유닛 사용
```
A가 메모리 대기 등으로 쉬는 타임을 B가 채운다.

### 한계점
두 스레드가 공유하는 자원이 있기 때문이다.
```
* 공유되는 것
실행 유닛, L1/L2 캐시, 메모리 버스

* 공유되지 않는 것
레지스터, 프로그램 카운터
```
두 스레드가 동시에 같은 실행 유닛을 원하면 결국 하나가 기다려야 한다. 위에서 실행한 예제는
정말 단순한 예제로 cpu 작업만 거의 하는 작업으로, 실행 유닛을 안 쓰는 틈이 거의 없어서 

### 정리
cpu가 어떤 이유로든 쉬는 틈이 생길 때 그 틈을 다른 스레드로 채우는 기술.
